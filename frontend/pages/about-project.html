Sobre o Projeto

O Sistema de Controle de Gastos Inteligente (SGCI) consiste em uma aplicação simples para gerenciar despesas e receitas do usuário, registrando-as no sistema. O software também permite estabelecer metas a serem cumpridas em um determinado período de tempo. Além disso, há a possibilidade de emitir relatórios com base em gráficos.
O projeto tem como foco a prática de técnicas de desenvolvimento Full Stack e metodologias ágeis. Vale ressaltar que o projeto não se propõe a ser algo inovador, genial ou visualmente sofisticado, mas sim um microprojeto desenvolvido ao longo de um bimestre.

Tecnologias e Ferramentas

Metodologia

No projeto, optou-se por utilizar o Scrum, metodologia que tem como foco o desenvolvimento ágil e a integração contínua, apresentando as seguintes características:

  • Scrum Master: Líder do grupo responsável por coordenar a equipe de desenvolvimento e auxiliar os demais membros.
  • Product Owner: Responsável por representar a ponte entre a equipe de desenvolvimento e o cliente.
  • Sprint: Ciclo de desenvolvimento que geralmente dura duas semanas.
  • Daily: Reunião diária para tratar das ações realizadas no projeto.
  • Sprint Review: Reunião ao final da sprint para debater o que foi feito.

Front-end

O front-end é a parte do site responsável por ser executada do lado do cliente, ou seja, a parte exibida no navegador com a qual o usuário interage. Dentre as tecnologias utilizadas estão:

  • HTML: HyperText Markup Language (Linguagem de Marcação de Hipertexto), tecnologia responsável por estruturar o conteúdo de uma página web.
  • CSS: Cascading Style Sheets (Folhas de Estilo em Cascata), linguagem que atribui estilo ao HTML, definindo regras de cor, alinhamento, espaçamento, tamanho, entre outros.
  • JavaScript: Linguagem de programação usada para adicionar interatividade ao site, com base em princípios de lógica de programação e manipulação do DOM (Document Object Model).
  • Bootstrap: Framework CSS comumente usado para auxiliar na estilização da aplicação, facilitando a criação de animações e padronizando características entre os componentes.
  • Chart.js: Biblioteca JavaScript desenvolvida para a criação de gráficos de forma simples e intuitiva.
  • Http-server: Pequena ferramenta de linha de comando usada para executar a aplicação por meio de um servidor local simples.

Back-end

O back-end é a parte que trabalha do lado do servidor, manipulando o banco de dados por meio de consultas e servindo dados ao front-end para que o usuário possa interagir. Para desenvolvê-lo, foram utilizadas as seguintes tecnologias:

  • Python: Linguagem de programação criada para ser simples e legível, utilizada no desenvolvimento da aplicação e das regras de negócio.
  • FastAPI: Framework Python criado para facilitar a criação de APIs. Seu ponto forte é o suporte a requisições assíncronas, o que melhora a performance da aplicação.
  • SQLModel: Framework que trabalha em conjunto com o FastAPI. É baseado em duas bibliotecas Python: SQLAlchemy e Pydantic.
  • SQLite: Banco de dados leve usado para persistir dados localmente. É conhecido por sua simplicidade, já que não depende de servidores — o banco é apenas um arquivo.
  • Uvicorn: Servidor ASGI utilizado pelo FastAPI para executar a aplicação.
